# BNO086 (BNO08x) IMU — compass + head tracking

The BNO086 is the CEVA BNO08x-family 9-DOF sensor. Unlike the BNO055 (simple
register map), it speaks **SH-2 / SHTP**, so ProtoHUD drives it via the vendored
CEVA SH-2 library (`vendor/sh2`) with an I2C + INT/RST HAL
(`src/sensor/bno08x.{h,cpp}`).

What it gives ProtoHUD, from the **ARVR-Stabilized Rotation Vector** (one report):
- **Compass heading** — magnetometer-referenced, so **no yaw drift** (the BNO055
  and the VITURE IMU both drift). Feeds `AppState::imu_bno08x`.
- **Head tracking** — AR/VR-jitter-stabilized roll/pitch/yaw for the `imu_pose`
  path (timewarp + pin-in-space), when `head_tracking` is enabled.

## Wiring (I2C)
| BNO086 | CM5 40-pin |
|---|---|
| SDA | GPIO2 (pin 3) |
| SCL | GPIO3 (pin 5) |
| VCC | 3.3 V |
| GND | GND |
| INT | any spare GPIO (data-ready, active-low) — strongly recommended |
| RST | any spare GPIO (active-low) — recommended |

I2C address is `0x4A` (default) or `0x4B` (SA0/ADR high).

## Enable
```json
"imu_source": "bno086",          // or "auto" (BNO086 is preferred when present)
"bno086": {
  "enabled": true,
  "i2c_bus": "/dev/i2c-1",
  "i2c_addr": 74,                 // 0x4A
  "gpiochip": "/dev/gpiochip0",
  "int_line": 5,                  // BCM offset of INT (-1 = poll, not recommended)
  "rst_line": 6,                  // BCM offset of RST (-1 = no hardware reset)
  "report_interval_us": 10000,    // 100 Hz
  "declination_deg": 0.0,
  "heading_offset": 0.0,
  "heading_invert": false,        // flip if the compass turns the wrong way
  "head_tracking": false          // true → BNO086 also drives head tracking
}
```

## Head-tracking notes
- With `head_tracking: true`, the BNO086 **takes over `imu_pose`** from the VITURE
  glasses' IMU (they no longer fight over the head orientation).
- Because it's a separate sensor (not auto-aligned like the in-glasses VITURE
  IMU), align "forward" with **GPIO → On-Board GPIO → IMU → Recenter Head Tracking** (issues an SH-2
  tare) after mounting or if the view drifts off-centre.

## Caveats
- **I2C clock stretching:** the BNO08x uses the INT handshake rather than heavy
  clock stretching, but if the bus is flaky, slow it
  (`dtparam=i2c_arm_baudrate=… ` in `config.txt`) or verify `i2cdetect -y 1`
  shows `4a`.
- Only **one** BNO08x can be active (the SH-2 library keeps process-global state).
- Stage 1 enables orientation only. Gestures (tap / shake / stability
  classifier) the BNO08x also provides are a planned follow-on.
